                     How to use personal API keys
                

                
            

            

                

                    Personal API keys let your own scripts, dashboards, and automations call protected Rolnopol APIs without borrowing your browser session.
                


                
    
                    
  1. 
                        Create a key for one integration.
                        Give it a clear label like Home Assistant sync so you know what it belongs to later.
                    
    2. 
                    
  2. 
                        Choose the smallest scope set possible.
                        If your tool only reads profile data, keep it on User account instead of giving it the whole barn.
                    
    3. 
                    
  3. 
                        Copy the raw key immediately.
                        The full value is shown only once after create or regenerate, so store it in your password manager, secret vault, or environment variables.
                    
    4. 
                    
  4. 
                        Choose an expiration that matches the risk.
                        Short-lived keys are great for temporary automations, while No expiration date is best reserved for integrations you actively monitor and rotate.
                    
    5. 
                    
  5. 
                        Send it with each request.
                        Your integration should include the header X-API-Key: your_key_here when calling protected endpoints.
                        How to use it?
    If you're using curl, add -H "X-API-Key: your_key_here" to your command. In JavaScript, include 'X-API-Key': 'your_key_here' in your fetch headers.
                    
    6. 
                    
  6. 
                        Rotate or revoke if anything looks suspicious.
                        Regenerate replaces the old secret with a new one, and revoke shuts the key down completely.
                    
    7. 
                


                

                    Important: key-management endpoints still require your normal signed-in session. API keys are meant for using protected APIs, not for creating, listing, or deleting keys themselves.
